## 重要度と優先度

### 重要度（影響の大きさ）
- [ ] S1: クリティカル（システム停止、データ損失、セキュリティ侵害）
- [ ] S2: 高（主要機能が使用できない、重要なビジネスプロセスの妨げ）
- [ ] S3: 中（機能は動作するが制限あり、操作性の大幅な低下）
- [ ] S4: 低（軽微な問題、視覚的な不具合、操作性の軽微な低下）

### 優先度（修正の緊急性）
- [ ] P1: 即時対応（本番停止、顧客影響大、回避策なし）
- [ ] P2: 高優先（次回リリース前に必須、顧客影響あり）
- [ ] P3: 中優先（計画的対応、回避策あり）
- [ ] P4: 低優先（都合の良いときに対応、影響が限定的）
